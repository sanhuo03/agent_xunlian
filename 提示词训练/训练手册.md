# 提示词工程实战训练手册

## 一、实验目标

本实验通过 DeepSeek 大模型和 Gradio WebUI，对比 6 种常见 Prompt 工程模式的效果。核心目标是理解：同一个大模型，在不同提示词和 temperature 参数控制下，可以产生不同风格、结构和稳定性的输出。

## 二、运行环境

- Python 3.11
- DeepSeek API
- Gradio WebUI
- OpenAI 兼容 SDK

`.env` 文件需要放在 `code/` 目录下，包含：

```env
DEEPSEEK_API_KEY=你的 DeepSeek API Key
DEEPSEEK_BASE_URL=你的 DeepSeek API 地址
```

## 五、训练内容

### 1. 裸调用

训练目的：观察不加特殊提示词时，大模型自由输出的状态。

输入示例：

```text
介绍一下 Python
```

预期输出：模型自由生成一段介绍，可能是段落，也可能是列表。

观察要点：

- 输出格式是否稳定
- 语气是否固定
- 多次运行结果是否完全一致

结论：裸调用最简单，但输出不可控，适合作为对比基准。

### 2. 角色扮演

训练目的：通过 System Prompt 控制模型身份、语气和表达风格。

输入示例：

```text
介绍一下 Python
```

角色设定示例：

```text
你是一位资深 Python 讲师，讲话幽默风趣，喜欢用生活化例子解释概念。
```

预期输出：模型会用教师口吻回答，表达更生动。

观察要点：

- 是否符合角色身份
- 是否体现指定语气
- 是否因为追求风格而减少有效信息

结论：角色扮演适合教学、客服、专家问答等需要固定语气的场景。

### 3. 结构化输出

训练目的：强制模型输出 JSON，让程序可以直接解析和处理。

输入示例：

```text
帮我比较 Python、Java、JavaScript 三种编程语言
```

预期输出：符合指定字段的 JSON 数据。

观察要点：

- `json.loads()` 是否能成功解析
- 是否出现 JSON 以外的多余文字
- 字段名是否符合要求

结论：结构化输出是 AI 与程序对接的重要方式，适合数据抽取、分类、表单生成等场景。

### 4. Few-shot

训练目的：通过 2 到 5 个示例，让模型模仿指定格式和回答方式。

输入示例：

```text
焦虑
```

示例逻辑：

```text
开心 -> 难过
快速 -> 缓慢
光明 -> 黑暗
```

预期输出：模型按照示例格式输出词语、反义词和解释。

观察要点：

- 是否模仿示例格式
- 是否比无示例时更稳定
- 示例数量是否足够

结论：Few-shot 不需要训练模型，只靠示例就能显著约束输出。

### 5. 思维链

训练目的：让模型分步骤推理，提升复杂问题的可检查性。

输入示例：

```text
小明有5个苹果，给了小红2个，又买了3个，然后吃掉1个，最后把剩下的一半给了小刚。小明还剩几个苹果？
```

预期输出：模型先列出推理步骤，再给出最终答案。

观察要点：

- 每一步推理是否正确
- 过程和结论是否一致
- 与直接回答相比是否更可靠

结论：思维链适合数学题、逻辑题、方案分析等需要推理过程的问题。

### 6. 格式约束

训练目的：给模型一个固定模板，让模型按模板填空。

输入示例：

```text
推荐3本 Python 入门书籍
```

预期输出：严格按照“书名、作者、难度、推荐理由、适合人群”的模板生成。

观察要点：

- 三条内容格式是否完全一致
- 是否有多余文字
- 模板是否便于阅读

结论：格式约束适合生成清单、推荐表、报告片段等给人阅读的内容。

## 六、temperature 实验

页面提供 temperature 滑块，范围为 0 到 1.0。

| temperature | 含义 | 适用场景 |
|---|---|---|
| 0 | 输出最稳定，随机性最低 | JSON、代码、严谨问答 |
| 0.7 | 稳定和变化较平衡 | 普通问答、解释、推荐 |
| 1.0 | 随机性最高，更有创意 | 文案、故事、头脑风暴 |

温度实验 Tab 会对同一个问题分别用 0、0.7、1.0 各运行 3 次，用于观察输出差异。

## 七、模式对比总结

| 模式 | 相对裸调用的优势 |
|---|---|
| 角色扮演 | 能控制身份、语气和风格 |
| 结构化输出 | 能让程序直接读取结果 |
| Few-shot | 能通过示例让模型模仿格式 |
| 思维链 | 能展示推理过程，便于检查 |
| 格式约束 | 能按固定模板稳定生成内容 |

## 八、实验结论

Prompt 工程的本质不是“随便问 AI”，而是用清晰的角色、规则、示例、格式和参数约束模型输出。提示词越具体，输出越稳定；任务越复杂，越需要结构化约束和验证机制。
