# 智能数据库查询 Agent 训练手册

## 一、任务目标

本项目实现一个可以理解自然语言的数据库查询 Agent。用户在 Gradio 聊天界面输入问题后，Agent 会调用大模型生成 SQL，再通过 `run_sql` 工具连接 SQLite 数据库并执行查询，最后把查询结果解释成中文回答。

项目重点不是写固定的 if-else 规则，而是让大模型根据用户意图和数据库结构自主决定 SQL。如果 SQL 执行失败，Agent 会把错误信息反馈给大模型，让它重新生成 SQL 并再次调用工具。

## 二、项目结构

```text
智能查询Agent作业/
├── .env              # DeepSeek 与数据库配置
├── requirements.txt  # Python 依赖
├── init_db.py        # 初始化示例数据库
├── students.db       # 运行 init_db.py 后生成
├── agent.py          # Agent 核心逻辑与 run_sql 工具
├── app.py            # Gradio 聊天界面
└── 训练手册.md        # 本说明文档
```

## 三、环境准备

进入项目目录：

```bash
cd 智能查询Agent作业
```

安装依赖：

```bash
pip install -r requirements.txt
```

如果使用虚拟环境，可以先执行：

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## 四、配置大模型

打开 `.env`，填写自己的 DeepSeek API Key：

```text
DEEPSEEK_API_KEY=你的真实Key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
DB_PATH=students.db
MAX_AGENT_TURNS=4
```

本项目使用 OpenAI 兼容 SDK 调用 DeepSeek，不需要本地部署模型。

## 五、初始化数据库

运行：

```bash
python init_db.py
```

程序会生成 `students.db`，其中包含三张表：

```text
classes(id, name, teacher)
students(id, name, age, gender, class_id, city)
scores(id, student_id, subject, score)
```

这些表可以支持单表查询、统计查询和多表 JOIN 查询。

## 六、运行 WebUI

运行：

```bash
python app.py
```

浏览器打开：

页面中可以直接输入自然语言问题，也可以点击示例问题测试。

## 七、核心代码说明

### 1. run_sql 工具

`run_sql(sql)` 是 Agent 可以调用的工具函数。它会连接 SQLite 数据库，执行大模型生成的 SQL，并返回 JSON 格式结果。

本项目默认只允许执行 `SELECT` 和 `WITH` 查询，避免误删或修改数据。执行成功时返回字段名、数据行和行数；执行失败时返回错误信息，供大模型下一轮修正。

### 2. 数据库结构可配置

`get_database_schema()` 会读取 SQLite 当前真实表结构，并放入 System Prompt。这样 Agent 不需要把表结构写死在某个固定函数里，后续如果更换数据库文件，也可以自动读取新表结构。

### 3. Agent Loop

`agent_loop()` 的执行流程如下：

```text
用户输入自然语言问题
↓
大模型读取数据库结构并生成 SQL
↓
调用 run_sql 执行 SQL
↓
如果 SQL 报错，把错误返回给大模型重新生成
↓
如果 SQL 成功，把查询结果整理成中文回答
```

终端会打印关键日志，包括生成的 SQL、工具执行结果和最终回复，便于验收时观察 Agent 的运行过程。

## 八、验收测试问题

可以测试以下问题：

```text
查询所有学生的姓名和年龄
年龄大于20岁的学生有几个？
查询每个班级的学生人数
查询 Python 成绩最高的前三名学生
查询软件一班学生的数据库成绩
查询数据库成绩大于90分的学生姓名、班级和成绩
```

也可以输入一个容易导致 SQL 错误的问题，例如：

```text
查询不存在的字段 average_age，看看 Agent 能不能修正
```

当 SQL 报错时，终端可以看到 `run_sql` 返回错误信息，Agent 会继续下一轮尝试修正。

## 九、运行效果说明

示例问题：

```text
年龄大于20岁的学生有几个？
```

可能生成的 SQL：

```sql
SELECT COUNT(*) AS count FROM students WHERE age > 20;
```

工具返回结果后，Agent 会回答类似：

```text
年龄大于20岁的学生共有5人。
```

这说明 Agent 已经完成了从自然语言到 SQL，再从数据库结果到自然语言回答的完整流程。
